Open-Source Information

Open-Source Projects

  • Community
    • Meritocracy
    • As a person makes valuable contributions (not necessarily code) they will be given or take on additional responsibilities
    • A person starts by pointing out a small change then making that change if it is accepted they begin to earn more recognition and progress towards making larger change and eventually become responsible for a portion of the project
    • Committer- a developer that is responsible for committing any changes to the code base submitted by others after they have reviewed them

  • Leadership
    • In most projects there is a “Benevolent Dictator” (“BD”) who have final say on decisions, they usually rely on other developers that are high up in the meritocracy to make decisions
    • As projects mature and grow they usually switch from BD to a democratic model where decisions are made by consensus

  • Forking
    • The ability for anyone to make their own copy of the code and make changes without affecting the base code
    • Example: MariaDB is a fork of MySQL

  • Communication
    • Wikis
    • IRC
    • Mailing lists
    • Bug trackers
    • Version control systems

  • Roadmaps
    • Lays out the future of the project
    • May be a detailed schedule of bug fixes and enhancements for larger projects or a list of the next enhancements for smaller projects

  • Repositories
    • Where projects save their code and users go to download the software

  • Packaging
    • Projects are distributed as source code meaning it must be downloaded, unzipped, untarred, and then installed using a manual process
    • Most projects create a binary package to make it easier to install
    • Pre-configured to be installed without having to build the project from the source code

  • Upstream/Downstream
    • Upstream- the main branch/version
    • Downstream- a copy developers work on
    • Changes made by developers are pushed upstream to the main branch

  • Version Control
    • Tracks and controls changes made to a project’s files

  • Trackers
    • Bug trackers or Issue trackers are used by most projects to provide a place to find, report, describe, prioritize, reproduce, diagnose, fix and test issues

  • Workflow
    • The method a project uses for getting code from contributors
    • Each project is different
