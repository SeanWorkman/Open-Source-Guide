Github

Github is a website that allows programmers a way to share and contribute code. A project's code is held in repositories, repoos, that are either public or private, free accounts are only capable of having public repos. Most open-source projects use github to store their source code and as an issue tracker.

  • Forking- makes a copy of the repository to your profile allowing you to edit it without changing the original
  • Cloning- exports the repository to your local computer
  • Committing- an changes made to the files on your computer are not automatically added to the repository on github you must commit the changes then push them for the code to be changed on github
  • Push- you must push all committed changes to the repository for them to be on github
  • Pull- to have your changes added to the original repository you must submit a pull request from your forked repository to the original which is reviewed and accepted if it is correct

Setup

  • Go to github
  • Create an account
  • Install git

Forking

  • Go to the repository you would like to change on github
  • Click the "Fork" button in the top right, this will add a copy to your profile

Cloning

  • Go to the forked repository on your profile
  • Copy the clone URL
  • Open the Git shell
  • cd to the directory you want to save the code to
  • Type git clone Clone_URL replacing "Clone_URL" with the one you copied

Syncing your Fork with the Upstream

  • Open the Git shell
  • cd to the directory
  • Type git remote -v this will list your remotes
  • Go to the github repository that you forked from and copy the clone URL
  • In the Git shell type git remote add upstream 'clone_URL'
  • Type git remote -v to make sure the upstream remote was added
  • Type git fetch upstream
  • Type git checkout whatever_branch_is_used to switch to that branch, the branch that is used for development changes depending on project, most use master (mifos uses develop) you can usually figure it out in the README.md file
  • Type git merge upstream/whatever_branch_is_used
  • Up until now only your local files are synced, to sync the fork on github you must push your changes
  • Type git push origin whatever_branch_is_used origin being the remote name you are pushing the branch to
Adding the Remote Syncing

Submitting Changes

  • Open the file you want to make changes to on your computer
  • After making the changes save the file
  • In the Git shell type git status to check if all files match the ones on github
  • To add the changes type git add . to add all files or git add name_of_file to add a specific file
  • Type git status again to make sure you have added all of the files you changed
  • Commit your changes, type git commit -m "a message describing what you changed"
  • Push your changes to the original repository, type git push origin whatever_branch_is_used and type your password if prompted
  • Go to your forked repository on github and click the pull request button

Git Commands

  • Fetch- git fetch will download the current state (containing updated and newly created branches) of an online repository without modifying your local repository. It places its results in .git/FETCH_HEAD.
  • Merge- git merge will merge the modifications of another branch into the current working branch.
  • Pull- git pull is a combination of git fetch and git merge. It fetches the information from an online repository's branch and merges it with your local copy.
  • Add- Whenever you modify a file in your local repository or create a new file, that file will appear as unstaged. Calling git add allows you to specify files to be added to your staging area.

Updating your Fork and Local Files

  • In your forked repository click "compare"
  • Compare your fork with the upstream, have your fork first then the upstream
  • If your fork is behind the upstream in commits you will see the "New Pull Request" button, click it
  • Submit the pull request, after the automatic checks are run a "Merge" button with appear click it to merge the repositories, this will update your fork on github
  • In the git shell, cd to the repository and type git pull, this will update your local files
